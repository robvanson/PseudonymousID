Create a pseudonymous ID

Create a pseudonymous ID based on a personal ID number and a combination of the Initials and the year of birth (Initials+YoB). An exhaustive search can find the original values matching the pseudonymous ID unless a strong Key is added for security. Therefore, make sure the personal ID number cannot easily be used to identify a person (more information).

All fields   ID Length
Initials+YoB  Personal ID  Pseudonymous ID 
 
Key (increases security)

You may paste a list of Initials+YoB/Personal ID pairs on separate lines in the text field. Initials+YoB and ID fields must be separated by commas, semicolons, or tabs. Only the first two fields are used, unless the All fields box is checked. With the All fields box checked, all text on a row is used to generate an ID. A "-" dash is inserted as a separator. A tab-delimited list of Initials+YoB, Personal ID, and Pseudonymous ID will be returned. Formats for Initials+YoB and Personal ID fields are not fixed. You can use any number and type of printable characters, except that you cannot use one of ", ;" (comma nor semicolon) nor tabs or newlines. Note that the fields are case sensitive "june", "June", and "JUNE" all give different Pseudonymous IDs.

The minumum length chosen for the pseudonymous ID should be at least 1 1/3 times the number of digits of the maximal number of person IDs ever needed (order of magnitude). For example, if upto around 100,000 persons would ever need a pseudonymous ID, a length of 8 would suffice (1.33*6). More precisely, there will be a % chance of two or more persons getting the same ID ("collision") after giving out 238 thousand ID numbers, for a chosen ID length of 8 characters (more information).

The Psuedonymous ID is created using the SHA512 algorithm (FIPS 180-2, see copyright notice below).
